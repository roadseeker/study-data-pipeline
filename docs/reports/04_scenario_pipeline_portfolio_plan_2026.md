# 2026년 Scenario Pipeline 포트폴리오 실행계획

이 문서는 3개월 역량 강화 계획 중 **scenario-pipeline** 영역의 상세 실행계획이다. 목표는 실제 데이터셋 6회 반복 실습을 통해 data-pipeline과 ml-pipeline 역량을 사업 제안 가능한 포트폴리오로 전환하는 것이다. 총괄 일정은 `01_capability_building_plan_2026.md`를 참조한다.

## 영역 C: 실제 데이터셋 기반 포트폴리오 6회 반복 실습 (4주 · 2026-06-22~2026-07-19)

데이터파이프라인과 AI/ML 구축 역량은 한두 개 예제를 완성하는 것만으로 충분히 검증되기 어렵다. 실제 고객 프로젝트에서는 데이터 원천, 품질 문제, 비즈니스 지표, 모델 적용 방식, 운영 결과 서빙 방식이 매번 달라지기 때문이다. 따라서 포트폴리오는 해외 데이터셋 3건과 국내 데이터셋 3건, 총 6회 반복 실습으로 구성한다.

6건 모두를 동일한 깊이의 대형 프로젝트로 만드는 것이 목표는 아니다. 핵심은 서로 다른 데이터 유형과 사업 문제를 대상으로 `수집 → 품질 검증 → Bronze/Silver/Gold 적재 → 피처 생성 → 모델 학습 → 예측 결과 저장 → API 또는 대시보드 서빙 → 운영 리포트 작성` 흐름을 반복 수행하는 것이다. 이 반복을 통해 특정 도구 사용 경험을 넘어, 고객사의 다른 데이터에도 같은 구축 패턴을 적용할 수 있는 실전형 역량을 확보한다.

각 실습은 최종 산출물의 형식을 통일한다. GitHub에는 데이터 파이프라인 구조, 실행 방법, 모델 평가 결과, 예측 결과 저장 구조를 정리하고, 블로그에는 비즈니스 문제 정의와 ROI 관점의 해석을 작성한다. 제안서 템플릿에는 "이 사례를 고객사 데이터에 적용하면 어떤 운영 지표를 개선할 수 있는가"를 연결한다.
