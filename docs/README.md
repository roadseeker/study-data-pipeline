# docs

문서 산출물을 역할별로 관리하는 폴더.

## 폴더 구조

- `guides/`: Week 1~8 실습 가이드
- `foundation/`: 환경 구성, 공통 운영, 기반 아키텍처 문서
- `kafka/`: Kafka 운영 가이드, 장애 보고서, Kafka 관련 문서
- `nifi/`: NiFi 개념, 수집 아키텍처, 스키마 표준화 문서
- `flink/`: Flink 스트림 처리 문서
- `spark/`: Spark ETL 및 데이터 이관 문서
- `airflow/`: Airflow 오케스트레이션 문서
- `reports/`: 컨설팅 리포트, 최종 보고서, 교차 주차 보고서

## 문서 배치 원칙

- 주차 가이드는 항상 `docs/guides/`에 둔다.
- 실제 산출물 문서는 해당 기술 도메인 폴더로 이동한다.
- 새 문서를 만들 때는 주차보다 역할과 기술 영역을 우선 기준으로 삼는다.
